# SEO Vision 👁

SEO Vision is a production-grade SaaS platform designed for website SEO analysis and performance auditing. It crawls target websites, evaluates core on-page parameters (headings, links, image tags, structured metadata, and response performance), computes individual metric scores, and provides print-friendly downloadable PDF summaries.

## 🚀 Key Features

*   **Audit Engine**: Performs automated, live crawls of any website.
*   **Metric Grading System**: Assesses individual category scores (Basic SEO, Technical SEO, Performance, Content, Image Alt tags, Link structure, and Structured metadata) alongside an overall score.
*   **Intuitive Visual Dashboard**: Displays averages and trends using dynamic HTML5 Canvas charts.
*   **Unified Navigation System**: Standardized sidebar grouping and responsive headers with instant appearance toggles.
*   **E2E PDF Exports**: Compiles print-friendly layouts and renders A4-sized PDF reports using Puppeteer.
*   **Secure Session Management**: Restrictive, token-based authentication (JWT) with secure storage.
*   **Dual-Theme Rendering**: Fully synchronized Light and Dark themes that transition instantly.

## 💻 Tech Stack

*   **Frontend**: HTML5, Vanilla JavaScript, CSS3 (Glassmorphism & Flex layouts).
*   **Backend**: Node.js, Express.js.
*   **Database**: MongoDB, Mongoose ODM.
*   **Auditing Utilities**: Cheerio (HTML parser), Puppeteer (PDF generator & crawler backup).
*   **Authentication & Security**: JWT (JSON Web Tokens), bcryptjs, Express Rate Limit, Helmet.

## ⚙️ Environment Configuration

Create a `.env` file in the project root:

```env
PORT=5000
MONGODB_URI=mongodb://localhost:27017/seovision
JWT_SECRET=your_jwt_secret_key_here
JWT_EXPIRES_IN=7d
NODE_ENV=development
```

## 🛠 Installation & Usage

### Prerequisites
*   Node.js (>=18.0.0)
*   MongoDB Instance

### 1. Clone & Install Dependencies
```bash
git clone https://github.com/your-username/seoVision.git
cd seoVision
npm install
```

### 2. Launch the Application
*   **Production Boot**:
    ```bash
    npm start
    ```
*   **Development Boot** (requires nodemon):
    ```bash
    npm run dev
    ```

The backend server boots on port `5000` (or `PORT` variable value). The frontend files in the `client/` folder can be served locally via any server (e.g. Live Server) or accessed directly in the browser.

## 🔑 Authentication Flow

1.  **Register / Log In**: Users submit credentials which are handled securely using `bcryptjs`.
2.  **JWT Issue**: A secure JWT token is generated by the server.
3.  **Client Cache**: Token is stored in the client's `localStorage` as `jwt_token`.
4.  **Route Protection**: Page scripts check for session tokens on load. Unauthenticated sessions are redirected to the Login view.

## 📄 API Route Overview

| Endpoint | Method | Authentication | Description |
| :--- | :--- | :--- | :--- |
| `/api/auth/register` | POST | Public | Registers a new user account |
| `/api/auth/login` | POST | Public | Authenticates credentials and returns a token |
| `/api/auth/profile` | GET | Protected | Retrieves current user details |
| `/api/auth/profile` | PUT | Protected | Updates current user details |
| `/api/analyze` | POST | Protected | Starts website crawl and returns audit scores |
| `/api/analyze/history` | GET | Protected | Retrieves paginated user audit logs |
| `/api/analyze/:id` | GET | Protected | Retrieves single audit record details |
| `/api/analyze/:id` | DELETE | Protected | Deletes an audit and its associated report |
| `/api/report/:id` | GET | Protected | Retrieves a detailed SEO score breakdown |
| `/api/report/download/:id`| GET | Protected | Renders and downloads a PDF format audit report |
